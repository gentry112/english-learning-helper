# 6.3整合 — 英语学习助手 详细说明

## 一、项目概述

**英语学习助手（六级 + 考研 整合版）** 是一款面向 CET-6 和考研英语一 备考学生的移动端网页工具。用户在一个页面内即可自由切换两种考试类型，每种考试类型拥有独立的模式体系、文件夹数据、AI Prompt 和历史记录。

- **页面标题**：英语学习助手
- **版本**：整合版（合并六级 v1.1 + 考研 v2.0）
- **访问方式**：浏览器直接打开 `index.html`，推荐手机端使用（微信内可直接打开）

---

## 二、顶层设计：考试类型切换

### 2.1 切换栏

页面最顶部是一行**渐变色切换栏**：
- 选中"📚 六级"→ 绿色渐变背景（`from-green-500 to-emerald-500`），白底绿字选中态
- 选中"📖 考研"→ 蓝色渐变背景（`from-blue-500 to-indigo-500`），白底蓝字选中态

切换考试类型后，会发生以下完整重置：
1. 页面标题 `<title>` 随之变化
2. 模式 Tab 栏重新渲染（六级3个 / 考研4个）
3. 重新加载对应考试类型的 localStorage 数据
4. 所有聊天消息清空，重新显示欢迎引导
5. 历史记录 Tab 列表重新渲染

### 2.2 数据隔离

| localStorage Key | 内容 | 说明 |
|---|---|---|
| `cet6-api-key` | DeepSeek API Key | 两种考试类型**共用**同一个 Key |
| `exam-type` | 当前考试类型 | `'cet6'` 或 `'postgrad'` |
| `cet6-data` | 六级全部数据 | 文件夹 + 历史记录 |
| `postgrad-data` | 考研全部数据 | 文件夹 + 历史记录 |

旧版数据（`cet6-folders`）在首次打开时自动迁移：根据 history 中 `type` 字段智能判断属于六级还是考研，迁移后删除旧 key。

## 三、考试类型配置（EXAM_CONFIG）

整合版的核心架构是**配置驱动**——两种考试类型的行为差异全部定义在 `EXAM_CONFIG` 对象中，模板和逻辑通过计算属性 `examConfig` 动态读取。

### 3.1 六级模式（3 个）

| 模式 Key | 界面标签 | 功能说明 |
|---|---|---|
| `vocab` | 📖 词汇查询 | 输入英文→释义/频率/扩展 |
| `translate` | 🈂️ 中译英 | 输入中文→解析+简单版+进阶版译文 |
| `essay` | 📝 大作文 | 议论文：题目解析→关键词→双版本模板 |

### 3.2 考研模式（4 个）

| 模式 Key | 界面标签 | 功能说明 |
|---|---|---|
| `vocab` | 📖 词汇查询 | 输入英文→释义/考研频率/扩展 |
| `en-to-cn` | 🈂️ 英译中 | 输入英文段落→解析+通顺版+精准版译文 |
| `short-essay` | ✉️ 小作文 | 应用文（书信/通知/备忘录）→解析+模板 |
| `long-essay` | 📝 大作文 | 图画/图表作文→解析+模板 |

### 3.3 配置内容项

每个考试类型的配置包含：
- `label` / `title`：显示名称和页面标题
- `storageKey`：独立的 localStorage key
- `frequencyLabel`：频率标注前缀（"六级"/"考研"）
- `modes`：模式定义（key、label、消息类型、输入校验规则）
- `historyTabs` / `historyTabLabels`：历史记录 Tab 列表
- `welcome`：每个模式的欢迎语
- `placeholder`：输入框占位符（含作文第二步的 keyword 占位符）
- `emptyHint`：各模式空状态提示语
- `keywordGuide`：作文关键词引导文案

## 四、核心功能详解

### 4.1 词汇查询（六级和考研共用）

**交互流程**：输入英文 → 发送 → AI 返回结果卡片

**结果卡片五分区**：

| 分区 | 内容 |
|---|---|
| 📝 拼写检查 | 正确则显示通过；有误则显示纠正建议（橙色高亮） |
| 📖 释义 | 短语/句子先整体释义（黄底），再逐词拆解（原形、变形、词性×释义、派生词） |
| 📊 频率 | 按考试类型标注"六级高频🔴"或"考研高频🔴"（中频🟡/低频🟢），附真题出现说明 |
| 📝 学习记录 | 首次查询 / "你已查询过该词，累计 X 次" |
| 📚 拓展 | 常考搭配、真题例句（含中文翻译）、近义词辨析、写作高级替换表达 |

**历史记录规则**：同一单词重复查询时 `count++`，最后一次的 AI 结果覆盖旧频率和释义。

### 4.2 翻译模式

**六级中译英**：中文 → AI 返回英文译文
- 📝 题目解析（句子结构 + 翻译思路）
- 📗 简单拿分版（2-3个英文译文，含得分档位 `🏆 10-12分/15分制`）
- 📘 进阶高分版（2-3个高级句式译文，含亮点 ✨ 和知识点 📚）

**考研英译中**：英文段落 → AI 返回中文译文
- 📝 题目解析（句子结构 + 翻译策略：拆分长句、语序调整等）
- 📗 简单拿分版（2-3个通顺中文译文，含得分档位）
- 📘 进阶高分版（2-3个精准译文，体现翻译功底）

**历史记录规则**：每次查询独立新建记录（不累加）。

### 4.3 作文模式（两步交互）

所有作文模式（六级大作文、考研小作文、考研大作文）均采用统一的两步交互：

**第一步：题目解析**
输入题目 → AI 返回解析卡片（类型判断、核心要点、建议结构/格式、避坑指南）→ 显示关键词引导

**第二步：模板生成**
输入关键词（可留空跳过）→ AI 返回双版本模板：

| 版本 | 背景色 | 特点 |
|---|---|---|
| 📗 简单版 | 绿色底 | 基础词汇句式，`[...]` 填空标记，按段落组织 |
| 📘 进阶版 | 蓝色底 | 高级句式地道表达，按段落组织 |

每个段落含四层信息：段落标签 → 英文模板 → 中文翻译 → 知识点说明

### 4.4 整合版新增：考研大作文结构化表单

考研大作文第一步解析后，**不显示普通关键词引导**，而是弹出一组结构化输入框：

**图画作文表单**（当 AI 判断为图画作文时）：
- 图中人物/主体
- 他们在做什么
- 反映什么寓意
- 额外关键词（可选）
- 按钮：`📝 详细生成` / `跳过`

**图表作文表单**（当 AI 判断为图表作文时）：
- 图表类型（柱状图/饼图/折线图/表格）
- 图表标题/主题
- 关键数据
- 整体趋势
- 反映什么现象
- 额外关键词（可选）
- 按钮：`📝 详细生成` / `跳过`

点击"详细生成"会将表单数据拼接后送入 AI 生成模板（`mode=detailed`）；"跳过"则不附加额外描述（`mode=skip`）。这解决了之前单独版本中"用户不知道如何描述图画"的问题。

## 五、文件夹与历史记录

### 5.1 文件夹管理

- 每个考试类型独立管理文件夹（六级文件夹和考研文件夹互不干扰）
- 默认创建"默认"文件夹
- 支持新建、切换、删除文件夹（至少保留一个）
- 切换文件夹清空聊天消息，重新加载欢迎引导
- 下拉菜单：当前文件夹 ✓ 标记，多余文件夹可删除

### 5.2 历史记录视图

点击导航栏 📊 进入历史记录页，Tab 数量随考试类型动态变化：

**六级历史 Tab**：📖 词汇 / 🈂️ 翻译 / 📝 作文（3个）
**考研历史 Tab**：📖 词汇 / 🈂️ 英译中 / ✉️ 小作文 / 📝 大作文（4个）

- 词汇 Tab：按综合权重降序（`权重 = 查询次数×0.4 + 频率分×0.6`，高频=3/中频=2/低频=1）
- 其他 Tab：按时间倒序
- 每条记录右侧 ✕ 支持单条删除
- 点击记录卡片 → 返回对话页，自动切换到对应模式并发起查询
- 顶部 🗑️ 支持清空全部记录（需二次确认）

## 六、技术架构

### 6.1 技术栈

| 层级 | 技术 |
|---|---|
| 页面框架 | Vue 3（CDN `unpkg.com/vue@3/dist/vue.global.prod.js`） |
| CSS 框架 | Tailwind CSS（CDN Play `cdn.tailwindcss.com`） |
| AI API | DeepSeek API（`deepseek-chat` 模型，前端 fetch 直调） |
| 数据存储 | localStorage |
| 文件结构 | **单文件 HTML**，无构建工具，无后端 |

### 6.2 自定义颜色

```css
wechat-green: #95EC69  /* 用户气泡 + 选中态 */
cet-orange:   #FF6B35  /* 高频标记 + 警告 */
bg-gray:      #F5F5F5  /* 聊天背景 */
text-dark:    #333333  /* 主文字 */
border-light: #E5E5E5  /* 边框 */
```

### 6.3 CSS 动画

- `.dot-1/.dot-2/.dot-3`：加载三点闪烁（1.4s 循环，依次 0.2s 延迟）
- `.shake`：输入框抖动（0.3s，空输入时触发）
- `.toast-enter-active/.toast-leave-active`：Toast 提示滑入/滑出
- `.modal-enter`：弹窗缩放入场（0.92→1）
- `.msg-enter`：消息上移淡入

### 6.4 Vue 应用状态

```javascript
state = {
  examType,           // 'cet6' | 'postgrad'
  currentView,        // 'chat' | 'summary'
  currentMode,        // 动态值域
  summaryTab,         // 动态值域
  folders,            // 当前考试类型的文件夹列表
  messages,           // 所有模式消息混存，按 mode 过滤
  essayContext,       // 统一的作文两步上下文 { topic, analysis }
  essayFormData,      // 考研大作文结构化描述表单
  // ...弹窗、Toast、确认框状态
}
```

### 6.5 AI Prompt 体系（共 6+ 套）

| Prompt | 适用考试 | 返回结构 |
|---|---|---|
| 词汇查询（六级版） | 六级 | spellCheck / definition / cetFrequency / extension |
| 词汇查询（考研版） | 考研 | 同上，角色和频率来源改为考研 |
| 中译英 | 六级 | analysis / basicVersions(en) / advancedVersions(en) |
| 英译中 | 考研 | analysis / basicVersions(zh) / advancedVersions(zh) |
| 作文题目解析（通用） | 两者 | topicAnalysis(type/keyPoints/structure/traps) |
| 作文题目解析（考研大作文） | 考研 | topicAnalysis(type/imageMeaning/keyArguments/structure/traps) |
| 作文模板生成（通用） | 两者 | basicTemplate / advancedTemplate / keywordsUsed |

所有 System Prompt 均要求 `response_format: { type: 'json_object' }`，temperature 固定 0.3，30 秒超时。

### 6.6 错误处理

| 错误码 | 提示 |
|---|---|
| 401 | ❌ API Key 无效，请前往设置更新 |
| 429 | ⏱️ 请求过于频繁，请稍后重试 |
| 网络错误 | ⏱️ 网络连接失败，请检查网络后重试 |
| 超时 (30s) | ⏱️ 请求超时，请检查网络后重试 |
| JSON 解析失败 | AI 返回格式异常，请重试 |
| 空输入（非作文第二步） | 输入框抖动 + 红字"请输入内容" |
| 词汇模式输入非英文 | "请检查输入，目前仅支持英文内容" |

## 七、两个独立版本与整合版的差异对比

| 维度 | 六级独立版 | 考研独立版 | 整合版 |
|---|---|---|---|
| 模式数量 | 3 个 | 4 个 | 动态 3/4 个 |
| 考试切换 | 不支持 | 不支持 | **顶部渐变色切换栏** |
| 数据存储 | `cet6-folders` | `cet6-folders` | `cet6-data` + `postgrad-data` + `exam-type` |
| 旧数据迁移 | 无 | 兼容 translate/essay→新 type | **自动迁移 `cet6-folders`** |
| 作文上下文 | 单个 `essayContext` | `shortEssayContext` + `longEssayContext` | **统一 `essayContext`** |
| 大作文表单 | 无 | 无 | **结构化描述表单**（图画/图表） |
| 翻译结果模板 | 独立模板 | 独立模板 | **共用模板**（`v.en \|\| v.zh` 自适应） |
| 作文解析模板 | 独立模板 | 独立模板 | **共用模板**（按字段存在性条件渲染） |
| 代码架构 | 硬编码 | 硬编码 | **配置驱动**（`EXAM_CONFIG`） |
| 页面标题 | 六级词汇翻译助手 | 考研英语一学习助手 | **英语学习助手** |
| 历史 Tab | 固定 3 个 | 固定 4 个 | **动态生成** |

## 九、使用方式

1. 用浏览器打开 `index.html`
2. 点击顶部渐变栏选择 **📚 六级** 或 **📖 考研**
3. 点击右上角 ⚙️ 设置 DeepSeek API Key（测试连接 → 保存）
4. 在对话输入框中输入内容，开始学习
5. 可通过 📂 文件夹下拉管理学习分组
6. 点击 📊 查看历史记录

## 十、限制与不在范围内

- 纯前端，无后端服务器
- 数据存于浏览器 localStorage，清除浏览器数据会导致记录丢失
- 依赖 DeepSeek API，需用户自行提供 API Key
- 无账号系统、无云端同步、无离线模式、无 PWA
- 不支持数据导出/导入
